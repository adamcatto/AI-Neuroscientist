supplying the whole task and task history does not really work, the agent seems to get stuck in a loop, where instead of addressing the problem from the previous cell, it keeps trying to address the initial task in the same way as it did originally, so if it encounters an error, it will keep making that same error.

One way around this is: in the while loop, have the model determine if there was an issue with the approach/code it just tried. If yes, then get it to debug ONLY that code and the reason why it failed to meet the criterion, and use the reasoning about the failure to fix the code in the next cell.

The other thing to try is a tree of thoughts. Maybe we can give the model an overarching task to guide its approach, but have it explicitly generate a list of subtasks to solve. For each subtask, check if that subtask can is a "leaf" node or a "composite" of multiple tasks. If leaf node, then write and execute the code cell; otherwise, create subtasks for the subtask in question, recursively until no more splits of the tree. Then can move on to the next task. For each leaf node, determine if the code adequately addresses the leaf node's task; if not, write a new cell utilizing the previous cell's code/output to better address the task. For each leaf node, pass the previous history of cell content (markdown/code/output) as input to the prompt.
